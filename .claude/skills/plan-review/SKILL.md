# プラン自己レビュー（ExitPlanMode の前に必ず実施）

プランを書いたら、**ExitPlanMode を呼ぶ前に**自分のプランを下記5観点で厳しくレビューし、
見つけた問題（Issues）を潰した改善版をプランファイルに反映してから提示する。
「問題なし」になるまで提示しない。

## 観点
1. **要件適合性** — ユーザーの要求を全て満たすか。勝手な仕様追加/変更をしていないか。不明点を推測で埋めていないか。
2. **技術的妥当性** — 現コードベースに適した方法か。既存アーキテクチャと整合するか。よりシンプルな実装はないか。
3. **影響範囲** — 修正対象以外への副作用はないか。既存機能を壊さないか。DB・API・画面・バッチへの影響を整理したか。
4. **不足事項** — 抜け漏れはないか。テスト観点は十分か。エラーハンドリングを考慮したか。
5. **実行可能性** — このプランだけで実装を開始できるか。不足情報があれば明確に列挙したか。

## 出力（ユーザーへ提示する形）
- **Review Result**: 問題なし / 要修正
- **Issues**: 問題点を箇条書き（致命/中/軽微の温度感も）
- **Improved Plan**: 修正をプランファイルへ反映（要点を本文でも要約）

## 運用ルール
- このレビューは**毎フェーズ・毎プランで標準実施**（CLAUDE.md「プラン自己レビュー（必須）」と同義）。
- 軽微でも気づきは Improved Plan に畳み込む。Issues を残したまま ExitPlanMode しない。
- スコープ境界（このフェーズで作る/作らない）の明記漏れは頻出 → 必ずチェック。
