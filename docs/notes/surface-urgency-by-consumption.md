# 緊急度は「消費の進捗」で見せ、緊急度で並べる

## 主張
一般に、ユーザーが「次に何をすべきか」を即断する必要がある一覧では、
**残量・消費の進捗（どれだけ使ったか／あと何日か）を視覚化し、緊急度の高い順に並べる**。
生の件数や名前の羅列より、「進捗バー＋締切までの距離」で見せた方が、開いた瞬間に優先順位が直感で伝わる。
判断の重い情報（価格など）は前面に出さず、詳細画面に委ねて一覧の動線を軽く保つ。

## 根拠となった経験
- [[decisions/2026-06-22-top-screen-stock-meter]] で、トップを在庫メーター型（残量バー＋緊急度ソート）に決定。チェックリスト型は情報量不足、カード型は価格前面で動線が重いとして却下した。

## 反例 / 例外
- [要確認] 進捗の概念がない一覧（純粋なタスクのトグル等）ではメーターは過剰。件数とチェックで足りる。
- 緊急度の算出根拠が曖昧/信頼できない場合、誤った並びでかえって判断を誤らせる。並べ替えの根拠（サイクル・next_order_date）が確かなときに有効。
