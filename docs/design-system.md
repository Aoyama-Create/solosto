# solosto デザインシステム（外観の真実）

> **外観の真実**は2つで一組:
> 1. ビジュアルの真実 = デザインモック [`docs/references/v2.1/solosto デザインモック (standalone) (1).html`](references/v2.1/) （ブラウザで開いて確認）
> 2. 文字に起こした正 = 本ファイル
>
> 実装はこのトークンを **Mantine テーマに一元写像**し、全画面で一貫させる。トークンとモックがズレたら本ファイルを更新する。
> 即参照は `design-system` スキル。ブランドの性格は**温かい・丸い・親しみやすい**（クリーム地＋オレンジ＋丸ゴシック）。

---

## 1. カラートークン

モック内の `COLOR` 凡例が名前を明示している（primary / 底値 / 切れそう / surface / ink）。

| トークン | 値 | 用途 |
|---|---|---|
| `primary` | `#F0883E` | ブランドオレンジ。主要アクション・アクティブ |
| `primary-deep` | `#E2752B` | hover／グラデ終端 |
| primary グラデ | `linear-gradient(135deg,#F0883E,#E2752B)` | 主要ボタン・ヒーロー |
| `success`（底値） | `#5BA672` | 底値・良好。緑 |
| `success-tint` | `#E8F2EA` | 緑の淡い面 |
| `alert`（切れそう） | `#E0654E` | 緊急・切れそう。赤橙 |
| `alert-tint` | `#FBE7E2` | 赤の淡い面 |
| `surface` | `#FBF7F1` | 基本の面（温かいクリーム） |
| 面バリエーション | `#FFF` `#FDFBF8` `#F3ECE2` `#F2EDE4` `#EFE9E0` `#EEE5D7` | カード・区切り・沈んだ面 |
| `ink` | `#3B342C` | 本文（濃茶） |
| ink 階調 | `#2B2722`（最濃）`#5C5347` `#8C8377` `#A89E90` `#B6ADA1` `#C9C0B4`（最淡） | 見出し〜補助テキスト・ボーダー |
| accent amber | `#F0B43E` | 補助アクセント |
| accent purple | `#9A7BC0` ／ tint `#ECDFF5` | カテゴリ等の識別 |
| peach tint | `#FCEBDC` `#FDF0E4` | オレンジ系の淡い面 |
| error | `#b00020` | 破壊的・エラー（控えめに） |

- **判断色の意味付け**: 底値=緑(`success`)、切れそう=赤橙(`alert`)。価格比較で「買い時！」等の判定ラベルは出さない（色とデータで示すに留める／システム定義 2.7）。
- 画像プレースホルダ: ストライプ `linear-gradient(45deg,#F3ECE2,#F3ECE2 5px,#EEE5D7 5px,#EEE5D7 10px)`。

## 2. タイポグラフィ（Google Fonts）

| 役割 | フォント |
|---|---|
| 見出し | **Zen Maru Gothic**（丸ゴシック） |
| 本文・UI | **M PLUS Rounded 1c**（丸ゴシック） |

- 両方とも日本語対応の丸ゴシック。weight は多用（数値強調・見出しは太め、補助は細め）。
- `preconnect` で `fonts.googleapis.com` / `fonts.gstatic.com`。日本語サブセットは自動（モックは unicode-range 分割の woff2 を同梱）。

## 3. シェイプ・エレベーション

- **角丸**: ピル `999px`（チップ・ボタン・タブ・バッジ）／カード `20〜28px`。全体に強い丸み。
- **影**:
  - カード（浮き）: `0 18px 50px rgba(60,52,44,.18)`
  - 微（区切り・小カード）: `0 1px 3px rgba(60,52,44,.05)` ／ `0 2px 8px rgba(60,52,44,.06)`
  - primary ボタンのオレンジグロー: `0 4px 12px rgba(240,136,62,.3)` ／ `0 8px 20px rgba(240,136,62,.32)`
- 影は黒ではなく**温かい茶 `rgba(60,52,44,*)`** を使う（クリーム地になじませる）。

## 4. コンポーネント原則

- **ボタン（primary）**: ピル、オレンジ塗り（またはグラデ）、白文字、オレンジグロー影。
- **ボタン（secondary）**: クリーム/白地・ink文字・薄ボーダー、ピル。
- **カード**: 白〜`#FDFBF8` 地、角丸 24px 前後、微影。情報は余白広めにゆったり。
- **チップ/タグ**: ピル、淡い面 + 該当色文字（カテゴリ・状態）。
- **メーターバー（Top の核）**: 消費サイクルの残量を横バーで可視化。緊急は `alert`、余裕は `success`/`primary` 寄り。
- **ボトムタブ**: 5項目（ホーム/商品/検索/通知/設定）。アクティブは primary。
- **アイコン**: `@tabler/icons-react`（丸みあるストロークアウトライン）。`stroke=1.6` 目安・`size` 16〜22・`currentColor` 追従（親の色＝アクティブ primary が乗る）。装飾用途は `aria-hidden`（アクセシブル名はラベル文字 / `aria-label`）。**絵文字は使わない**（OS依存・色追従しないため）。
- **数値表現**: 単価 `¥0.8/個`、底値・前回比などは色（success/alert）と併記。

## 5. レイアウト / ナビゲーション

- **モバイルファースト PWA**（iPhone Safari がメイン）。`env(safe-area-inset-*)` でノッチ／ホームインジケーターを確保。
- **ボトムタブ5**: ホーム / 商品 / 検索 / 通知 / 設定。
- **PC版**（左サイドバー＋テーブル）は**後続フェーズ**（実装ロードマップ Phase 8）。

## 6. Mantine テーマ写像方針（実装の指針）

`lib/theme.ts`（パス暫定）で `MantineProvider` に渡すテーマへトークンを一元写像する。Mantine デフォルトの青・フォントは**使わない**。

- `primaryColor`: 独自オレンジスケール（`#F0883E` 基点の10段）を `theme.colors` に登録し指定。
- `theme.colors`: `success`（#5BA672基点）/ `alert`（#E0654E基点）の10段も登録。
- `fontFamily`: `'M PLUS Rounded 1c', sans-serif`。`headings.fontFamily`: `'Zen Maru Gothic', sans-serif`。
- `defaultRadius`: `md`（カード相当）。ピルは `radius="xl"`/`9999`。
- `theme.other`: surface 階調・ink 階調・影トークン・グラデを保持し、全コンポーネントから参照。
- `components`: `Card` / `Button` / `Badge` の default props（radius・shadow・variant）でモックの見た目を既定化。
- グローバル: `body` 背景 `surface(#FBF7F1)`、文字 `ink(#3B342C)`。

## 7. 画面別レイアウト要点（モック → SCR）

すべての画面がモック済み。**視覚の最終確認はモックHTMLを開いて差分を詰める**。

- **SCR-030 トップ（買うもの）= ★案C「在庫メーター型」を採用**。
  - 日付ヘッダ → 「そろそろ切れるもの N件」「9:41 に最新の状態を取得」。
  - グループ見出し「もう切れてる・切れる直前」。各行に商品名・残り日数（例「あと2日」「予定を3日超過」）・「サイクル 約45日」・**消費残量メーターバー**・ECディープリンク（Amazon↗/楽天↗/公式↗）・「買った！」アクション。
  - 銘柄横断（カテゴリ追跡）行: 「ワイン全体で約30日 · 前回 OSCO」「銘柄を選ぶ」。
  - **緊急度で並べ替え**。空状態（0件）=「今は買うものなし／ストックは十分そろっています」を肯定的に。
  - 案A（チェックリスト）・案B（カード）は**不採用＝参考残置**。
- **SCR-002 サインイン**: メール＋パスワード、「表示」トグル、新規登録導線。コピー「『そろそろ切れる』を、もう忘れない。」
- **SCR-010 商品一覧**: カテゴリ chips（すべて/日用品/食品/コスメ…）、各行に「約45日サイクル/あと18日/在庫あり/そろそろ/ミュート中」、銘柄横断は「ワイン · 銘柄横断 · 前回 OSCO」。
- **SCR-011 商品登録**: 「名前だけで即登録も可」、購入単位・標準入数・種別。定期にすると2回目から学習する旨の説明。
- **SCR-012 商品編集**: 種別（定期/単発）、次回購入サイクル（自動 現在約45日 / 手動 [45]日）、スヌーズ、通知ON/OFF、購入先URL。
- **SCR-021 購入登録モーダル**: 金額・購入日・パック数・入数 →「総個数 12個（自動計算）」「単価 /個」。category追跡品は銘柄も記録。
- **SCR-022 価格比較ビュー**: 底値/個（記録日）・平均/個（全N回）・直近の購入・「単価の推移」折れ線（●底値/●直近）。**判定ラベルは出さない**。
- **SCR-031 商品検索**: 名前・カテゴリ・プラットフォーム・期間の chips、結果件数。
- **SCR-040 通知センター**: 「買うもの N件」＋商品名羅列、相対日時、既読/未読、「すべて既読」、「1週間より前は自動削除」。
- **SCR-003/006 通知設定**: 「このデバイスで通知を有効にする」、デバイス一覧（iPhone=有効/MacBook 最終受信）、通知時刻（1日1通）、タイムゾーン（Asia/Tokyo）。
- **SCR-015 カテゴリ追跡設定（v2.1）**: 「商品単位／カテゴリ単位（銘柄横断）」トグル、このカテゴリの通知、サイクル（自動 約30日）。「追跡方法を変えると未購入分はリセット」の注意。
- **SCR-023 銘柄選択（買い足し）（v2.1）**: 履歴の銘柄一覧（OSCO 前回 6/15 ¥1,180 公式 ↗ / MAPU 6/1 ¥1,280 Amazon ↗）、「別の銘柄を買う」、戻って「買った」で選んだ銘柄を記録。
