# solosto 実装ロードマップ（個人用MVP）

> 別プランでの実装はこの順に進める。各フェーズは「依存が少ない→多い」順。
> 機能IDの真実は [機能一覧](references/v2.1/機能一覧_v2.md) / [マトリクス](references/v2.1/マトリクス_v2.md)。仕様は [システム定義書](references/v2.1/システム定義書_v2.md)。
> ドメイン式・状態遷移は `domain-rules` スキルと [CLAUDE.md](../CLAUDE.md) の不変条件を必ず参照。
> **外観はデザインモック準拠**。トークン正は [design-system.md](design-system.md)（即参照 `design-system` スキル）、ビジュアルはモックHTML（references/v2.1）。
> 進め方: 各フェーズで適切な subagent（supabase-schema / domain-logic / server-action / mantine-ui / pwa-push）に委譲。UIは design-system.md / モックに準拠。
