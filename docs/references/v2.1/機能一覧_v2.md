# 機能一覧 v2（個人用MVP）

> 設計書v2に対応。グループ共有系（招待・ユーザーメンテ）は「MVP対象外」へ分離。個数ベースのサイクル、商品3軸フラグ、スヌーズ、バッジ、送信ログ等を追加。

凡例：SCR=画面 / API=API（Server Action） / COM=共通 / BAT=バッチ

---

| SCR-002 | サインイン画面 | SCR | ログイン |

## v1からの主な変更点

- **分離**: 上記「MVP対象外」を本体から切り出し。
- **追加**: SCR-006（通知時刻設定）、API-016〜018（手動投入/スヌーズ/サイクル上書き）、API-042（通知作成）、COM-015（状態遷移）、COM-042〜043（失効検知/バッジ）。
- **変更**: BAT-001 を毎時実行へ。COM-021 をper_unit換算へ。COM-020 を total_units 基準へ。API-030 が件数も返す。

## v2.1での変更点（追跡単位 / カテゴリ単位サイクル）

- **追加**: SCR-015（カテゴリ追跡設定）、SCR-023（銘柄選択UI）、API-019（カテゴリ追跡設定更新）、API-023（銘柄一覧取得）、COM-016（scope分岐解決）。
- **置き換え**: SCR-014→SCR-015、API-015→API-019、COM-013→COM-016（target_type運用を tracking_scope に置換）。
- **scope対応**: SCR-013/030、API-014/016/017/021/030、COM-021/022/050 に「商品単位/カテゴリ単位の出し分け」を反映。
- **DB**: categories に tracking_scope 等を追加、purchase_logs に brand/purchase_url を追加。
